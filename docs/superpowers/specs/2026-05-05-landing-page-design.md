# 知行录官网设计文档

## 背景与目标

知行录目前没有公开的产品介绍页面。用户只能通过 GitHub README 了解产品。需要一个独立的官网落地页来：

- 向潜在用户清晰传达产品价值
- 展示三大核心功能的实际效果
- 提供快速开始的部署指南
- 建立品牌信任感

## 设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 页面形式 | 单页落地页 | 内容量适中，单页滚动体验流畅 |
| 内容策略 | 叙事驱动型 | 目标用户是散户投资者，情感共鸣比功能罗列更有效 |
| Hero 布局 | 居中标题 + 下方截图 | 经典 SaaS 落地页模式，简洁大气 |
| 视觉风格 | 沿用现有品牌 | 保持一致性，暖色调编辑风格 |
| 部署方式 | 独立目录，静态部署 | 可部署到 GitHub Pages / Vercel，与主应用解耦 |

## 技术方案

### 目录结构

```
website/
├── index.html          # 主页面
├── css/
│   └── style.css       # 样式文件
├── js/
│   └── main.js         # 交互逻辑（滚动动画、导航）
└── images/             # 产品截图等静态资源
    ├── hero-screenshot.png
    ├── feature-analysis.png
    ├── feature-letter.png
    └── feature-diagnosis.png
```

### 技术栈

- 纯静态 HTML/CSS/JS，无构建步骤
- CSS 自定义属性复用现有设计系统
- IntersectionObserver 实现滚动渐入动画
- 响应式设计（移动端断点 768px）

## 视觉规范

### 配色

| 用途 | 色值 | 说明 |
|------|------|------|
| 页面背景 | `#FAF7F2` | 暖白，纸张质感 |
| 主色 | `#2C3E2D` | 深森林绿，导航/标题/按钮 |
| 点缀色 | `#C9A961` | 暖金色，标签/高亮 |
| 正文 | `#2A2A2A` | 深灰 |
| 次要文字 | `#6B6B6B` | 中灰 |
| 边框 | `#E5E0D8` | 暖灰边框 |
| 卡片背景 | `#FFFFFF` | 纯白 |

### 字体

- 标题/品牌：`'Noto Serif SC', serif`
- 正文：`'Noto Sans SC', system-ui, sans-serif`
- 代码：`'IBM Plex Mono', monospace`

### 间距与圆角

- 区块间距：80px（桌面）/ 48px（移动）
- 卡片圆角：12px
- 按钮圆角：8px
- 卡片阴影：`0 2px 8px rgba(0,0,0,0.06)`

### 动效

- 滚动渐入：元素进入视口时从下方 20px 淡入，duration 0.6s
- 导航：滚动后添加底部阴影
- 按钮 hover：轻微上移 + 阴影加深
- 保持克制，不使用花哨动画

## 响应式策略

| 断点 | 布局调整 |
|------|----------|
| > 1024px | 完整桌面布局，左右分栏 |
| 768px - 1024px | 缩小间距，保持分栏 |
| < 768px | 单列布局，图片在文字下方，导航变汉堡菜单 |

## 验证方式

1. 本地打开 `website/index.html` 检查页面渲染
2. 使用浏览器开发者工具测试响应式布局（桌面/平板/手机）
3. 检查滚动动画是否流畅
4. 验证导航锚点跳转正确
5. 确认所有链接可点击（GitHub 等）
6. Lighthouse 检查性能和可访问性
