# 更新日志

## v1.3.0 (2026-05-18)

### 新增
- **网页端 LLM 配置面板**：首页标题旁新增齿轮图标，用户可在浏览器中直接配置 API 地址、API Key 和模型名称，无需编辑 .env 文件或重启服务
- 新增 `user_settings` 表（SQLite），持久化存储用户自定义 LLM 配置
- 新增 3 个 Settings API：`GET/PUT/DELETE /api/settings`，支持读取、保存、重置配置
- 支持 5 种预设 API 地址快捷参考：OpenAI、阿里云 DashScope、DeepSeek、智谱、Ollama 本地

### 优化
- `llm_client.py` 重构：去除全局单例 client，改为 `_resolve_config()` 动态解析，支持运行时热切换配置
- 配置优先级：网页用户设置 > .env 环境变量，保存后立即生效，无需重启服务
- API Key 输入支持密码模式（显示/隐藏切换），GET 接口返回脱敏后的 key
- 修改模型或 Base URL 时不改动 API Key，自动保留原值

## v1.1.2 (2026-05-11)

### 新增
- 模块8新增**自定义混合策略表单**：支持多账户、多策略（短线/中线/长线）并行配置，用户可填写账户规模、风险偏好、仓位比例等参数后提交，AI实时流式返回个性化执行方案
- 新增独立流式API `/api/mixed-strategy/{symbol}`，用户提交表单后可单独获取混合策略分析，不影响主报告生成流程

### 修复
- 修复模块2数据获取崩溃导致整个报告流中断的问题：akshare财务数据列名从「销售毛利率」变更为「销售净利率」，全局对齐8处引用
- 修复自定义策略表单不渲染的问题：之前表单HTML被嵌入LLM Prompt中，经 `_md()` 函数转义后变为纯文本，改为在生成器中直接拼接原始HTML
- 修复Module 0（操作建议）显示空白的根因：模块2崩溃后流式响应提前终止，模块0内容未完整传输到前端

### 其他
- 新增 `run_noproxy.py` 启动脚本，强制清除代理环境变量，避免akshare请求东方财富API被代理拦截

## v1.1.1 (2026-05-11)

### 新增
- **模块8：三种投资策略对比**（短线 / 中线 / 长线），从适合人群、资金规模、持仓周期、风险等级多维度给出具体操作方案
- Module 0（操作建议）全面升级为分步骤执行计划：买入分三步（首次建仓 / 下跌加仓 / 突破追入），卖出分两步（目标价分批止盈 / 止损清仓），包含具体价格区间和仓位比例

### 优化
- 模块0 Prompt重写：从简单的一句话态度升级为包含**具体操作步骤、关键时间节点**的可执行方案
- 交易参考模块（trade_ref）Prompt升级：增加合理交易区间（买入/卖出区间）的详细论证，要求每段都有具体数据支撑

### 修复
- 仪表盘（gauge）重叠bug：调整了指标卡片的渲染配置，避免多个指标重叠显示
