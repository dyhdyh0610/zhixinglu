# 知行录 UI/UX设计规范

> 摘自《知行录 产品需求文档》v1.1 第7章,并扩展跨端设计策略

---

## 1. 设计哲学

**核心原则**:像一份高级研报,不像一个交易终端。

**反例**:满屏跳动的红绿数字、弹窗式的"今日热门股"、煽动性文案、大V网红头像、"快速买入"按钮

---

## 2. 视觉风格

### 2.1 色彩系统

**主色调**:
- 主色:**深墨绿** `#2C3E2D` 或 **深蓝灰** `#2A3B4D`
- 强调色:**暖金色** `#C9A961` 或 **淡橙** `#E8A87C`(用于关键数据和操作按钮)
- 背景:**米白** `#FAF7F2`(护眼,长时间阅读不累)
- 文字:**深炭灰** `#2A2A2A`(主文字)、`#6B6B6B`(辅助文字)

**涨跌色**(关键差异化):
- 涨:**淡红** `#D97757`(饱和度低)
- 跌:**淡绿** `#7A9B6E`(饱和度低)
- **不用刺激的红绿对比色**

### 2.2 字体系统

**中文**:
- 标题:**思源宋体** 或 思源宋体Variable(让产品有"读物感")
- 正文:**思源黑体** 或 系统默认无衬线字体
- **宋体的使用是关键差异化**,绝大多数股票产品都用黑体

**英文/数字**:
- 标题:衬线体(如 Source Serif Pro)
- 数据:等宽字体(如 IBM Plex Mono)

**字号**:
- 偏大,行距偏宽(行距1.6-1.8)
- 鼓励"慢读"而不是"扫读"

### 2.3 图表风格

- 简洁,无多余装饰
- 不用3D饼图、不用花哨动画
- 数据为王,排版克制
- 折线图用细线,色块用低饱和度

---

## 3. 交互原则

### 3.1 信息密度低,留白多

- 一屏只解决一件事
- 反对信息瀑布流,反对一屏堆20个数据
- 用户应感觉"清爽"和"我需要思考",而不是"我需要赶紧操作"

### 3.2 没有红绿闪烁

- 行情数据更新不用闪烁
- 涨跌色块用浅色
- 数字变化用淡入淡出,不用跳动

### 3.3 操作有摩擦

**关键操作有二次确认**:
- 发布决策(公开)
- 公开持仓
- 删除决策日志

**不是因为想麻烦用户,是因为想让用户认真对待。**

### 3.4 文案有温度

**不要**:"系统繁忙,请稍后再试"
**要**:"AI正在认真分析你的持仓,大约还需要20秒"

**整个产品的文案风格**:克制、真诚、像朋友说话,**不打鸡血**。

## 4. 关键页面布局

### 4.1 首页(底部Tab "首页")

**展示内容(优先级从上到下)**:
1. 今天的提醒卡片:
   - 你昨天的交易需要记录决策日志吗?
   - 你哪些决策的核心假设可以验证了?
   - 你的Agent今天有什么想和你说的?
2. 你的持仓今日表现简报(基于个人持仓)
3. 今日市场对你持仓的影响(个性化,不是市场大盘)
4. 你关注的用户的最新决策(前3条)
5. 入口:单股分析、持仓诊断

**不展示**:
- ❌ 大盘行情
- ❌ 热门股票推荐
- ❌ 营销banner

### 4.2 我的(底部Tab "我的")

**3个子Tab**:
- 决策日志(默认)
- 持仓
- 复盘报告

### 4.3 Agent(底部Tab中央按钮,圆形突出)

- 圆形按钮设计,视觉锚点
- 进入后是对话流页面
- 对话历史在左上角入口

### 4.4 社区(底部Tab "社区")

- 顶部3个Tab:关注 / 精选 / 新发
- 内容流是决策卡片帖
- 右下角悬浮按钮:"分享一条决策"

## 5. 决策卡片视觉模板(全产品视觉锚点)

**决策卡片是全产品的核心视觉元素**,在以下场景使用同一种视觉模板:
- 决策日志列表
- 社区帖子主体
- 月度复盘报告
- Agent对话引用

**卡片包含**:
- 顶部:操作类型徽章(买入/卖出/加仓/减仓,不同颜色低饱和度)
- 中部:标的名称+代码、价格、占比
- 核心区:**核心假设**(用大号字突出,加引号)
- 底部:决策日期、当时的情绪图标
- 右上角(动态回填):距今天数、当前盈亏

**让用户产生"看到这种卡片就知道是认真的决策"的认知。**

## 6. 跨端设计策略(Web + 移动端并重)

### 6.1 总体原则

产品需要在 **Web端和移动端(iOS + Android)同时提供完整体验**,不分主次。两端共享同一套设计语言和交互逻辑,但根据各自平台特性做适配。

### 6.2 技术方案建议

- **推荐响应式Web应用(PWA/SPA)**:一套代码同时覆盖桌面浏览器和移动浏览器,降低开发和维护成本
- 前端框架建议:React / Vue + 响应式CSS框架(如Tailwind CSS)
- 如后续需要原生能力(推送、OCR等),可通过 **React Native / Flutter** 封装原生壳,复用Web核心逻辑
- 图表库统一使用 ECharts 或 Chart.js,两端渲染一致

### 6.3 响应式断点设计

| 断点 | 宽度范围 | 适用场景 | 布局策略 |
|------|---------|---------|---------|
| Mobile | < 768px | 手机竖屏 | 单列布局,底部Tab导航,卡片全宽 |
| Tablet | 768px - 1024px | 平板/小屏笔记本 | 双列布局,侧边导航可折叠 |
| Desktop | > 1024px | 桌面浏览器 | 三栏布局(导航+内容+辅助面板),最大内容宽度1200px |

### 6.4 移动端适配要点

- 底部Tab导航(首页 / 社区 / Agent / 我的),Agent按钮居中突出
- 所有交互优先考虑**单手操作**,关键按钮在拇指热区
- 决策日志填写考虑**通勤场景**:大按钮、少打字、多选项
- 图表支持手势操作:双指缩放、左右滑动切换时间区间
- 持仓截图OCR使用设备本地能力
- 推送通知走系统原生通道(APNs / FCM)

### 6.5 Web端适配要点

- **侧边栏导航**替代底部Tab,左侧固定导航栏
- 充分利用大屏空间:
  - 单股分析报告:左侧目录导航 + 右侧内容,可快速跳转模块
  - 决策日志:左侧列表 + 右侧详情,无需反复切换页面
  - Agent对话:左侧历史列表 + 右侧对话流,类似ChatGPT桌面布局
  - 月度复盘:沉浸式阅读模式,居中单列,最大宽度800px
- 支持**键盘快捷键**:
  - `/` 聚焦搜索框(单股分析)
  - `N` 新建决策日志
  - `←` `→` 切换日志条目
- 持仓导入支持**拖拽上传截图**和**手动输入表格**
- 社区帖子支持**悬浮预览**(hover显示决策卡片摘要)

### 6.6 两端体验一致性要求

- **数据实时同步**:用户在手机记录的决策,Web端立即可见,反之亦然
- **视觉一致**:色彩、字体、卡片样式、图表风格两端统一,不因平台不同而产生割裂感
- **功能对等**:所有核心功能(单股分析、持仓诊断、决策日志、复盘、社区、Agent)两端均完整可用,不做功能阉割
- **状态同步**:阅读进度、Agent对话上下文、未读提醒等状态跨端同步

### 6.7 各功能的跨端布局差异

| 功能 | 移动端 | Web端 |
|------|--------|-------|
| 单股分析 | 单列长页面,逐模块滚动 | 左侧模块目录 + 右侧内容,可锚点跳转 |
| 持仓诊断 | 截图拍照/相册上传 | 拖拽上传截图 + 手动输入表格 |
| 决策日志 | 全屏分步填写(3步) | 单页表单,左侧列表右侧编辑 |
| 月度复盘 | 全屏沉浸式,逐章节翻页 | 居中单列阅读,侧边章节导航 |
| 社区 | 信息流卡片,上下滑动 | 双列瀑布流 或 列表+详情分栏 |
| Agent | 全屏对话流 | 左侧历史列表 + 右侧对话,可调整宽度 |
